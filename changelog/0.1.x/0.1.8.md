# 0.1.8 — 2026-05-24

## Changed

- **`GhoService.normalizeRow`** — object literal with spread over sequential mutation; eliminates the intermediate `row` variable.
- **`GhoService.listDimensionValues`** — same object-literal-with-spread refactor.
- **`GhoService.queryData`** OData error guard — `'error' in data` replaces `'error' in data && data.error`; the truthiness check was redundant.
- **`who-indicator-metadata` resource handler** — `getIndicatorDimensions` and `listIndicators` now run via `Promise.all` instead of sequentially.
- **`who-get-indicator-metadata` tool handler** — service accessor hoisted to a single `svc` local; inner type guard removed (result type already narrows correctly after `Promise.all`).
- **`ambiguous_spatial_filter` error contract** — corrected from `JsonRpcErrorCode.InvalidParams` to `JsonRpcErrorCode.ValidationError` to match framework conventions for input-validation failures.
- **`@cyanheads/mcp-ts-core`** ^0.9.7 → ^0.9.9 (fuzzTool pre-parse fix, ZodArray minLength fix, `fast-check` moved to optional peer dep, "Close the loop on issues" rule added to CLAUDE.md).
- **`skills/git-wrapup`** — v1.0 added (synced from framework).
- **`skills/release-and-publish`** — v2.5 → v2.6 (branch-before-tags push order, GHCR auth hint, MCP Registry version lookup URL, idempotent re-invocation note).
- **`CLAUDE.md`** — "Close the loop on issues" rule added; `git-wrapup` skill entry added to skill table; framework version updated to `^0.9.9`.
- **`scripts/check-framework-antipatterns.ts`** — file header clarified to document consumer-side rationale (synced from framework).
