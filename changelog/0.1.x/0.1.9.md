# 0.1.9 — 2026-05-28

## Changed

- **`@cyanheads/mcp-ts-core`** ^0.9.9 → ^0.9.13 — picks up:
  - **`MCP_HTTP_MAX_BODY_BYTES`** — configurable inbound request-body cap (default 1 MiB); oversized requests rejected with 413 before the SDK parses the body.
  - **HTTP session-init gate** — stateful HTTP mode rejects non-`initialize` requests missing `Mcp-Session-Id` with 400, preventing uninitialized session minting.
  - **`httpErrorHandler` log-level split** — expected client errors (401, 403, 400, 404) downgraded to `warning` instead of full error-pipeline with stack traces, reducing noise.
  - **`GET /mcp` keywords** — `package.json` `keywords` now surfaced on the status endpoint alongside `name`, `version`, and `description`.
- **`@biomejs/biome`** ^2.4.15 → ^2.4.16.
- **Skills synced from framework** — `api-canvas`, `api-config`, `design-mcp-server`, `git-wrapup`, `polish-docs-meta`, `release-and-publish`, `report-issue-framework`, `code-simplifier` (new skill added).
- **`skills/migrate-mcp-ts-template/`** — removed (migration era complete; dropped from framework in 0.9.12).
