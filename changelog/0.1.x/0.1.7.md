# 0.1.7 — 2026-05-24

## Fixed

- **OData error envelope** — `queryData` in `GhoService` now detects `{"error": {...}}` responses from the GHO API and throws `serviceUnavailable` instead of crashing with a `TypeError` on the missing `@odata.count` / `.value` fields. ([#3](https://github.com/cyanheads/who-gho-mcp-server/issues/3))
- **`indicator_not_found` recovery hint** — the `who_query_indicator_data` handler now re-fails service-layer `indicator_not_found` errors via `ctx.fail`, ensuring the typed error contract's `recovery.hint` reaches the client. ([#4](https://github.com/cyanheads/who-gho-mcp-server/issues/4))
