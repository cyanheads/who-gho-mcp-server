# 0.1.11 — 2026-06-02

## Changed

- **`@cyanheads/mcp-ts-core`** ^0.9.16 → ^0.9.21 — HTTP transport per-request log context fix (per-request logs and traces now carry fresh request + trace/span IDs instead of the frozen boot context); `fetchWithTimeout` strips query-string secrets (e.g. `?api_key=`) from error messages and logs; `withRetry` fails fast on non-retryable errors and `ctx.fail` auto-populates the `retryable` flag.
- **Skills synced from framework** — `add-tool`, `add-service`, `api-canvas`, `api-context`, `api-linter`, `api-utils`, `design-mcp-server`, `release-and-publish`; new: `api-mirror`, `orchestrations`.
- **`scripts/devcheck.ts`** — refreshed from framework template.
- **`scripts/check-skill-versions.ts`**, **`scripts/release-github.ts`** — new framework-provided scripts; `release:github` script added to `package.json`.
- **README client-config key** renamed from `who-gho` to `who-gho-mcp-server` (full package name) in all three install examples (bunx, npx, Docker).

## Dependencies

- `@cyanheads/mcp-ts-core` ^0.9.16 → ^0.9.21
- `vitest` ^4.1.7 → ^4.1.8
