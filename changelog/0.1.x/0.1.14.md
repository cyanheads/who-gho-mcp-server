# 0.1.14 — 2026-06-20

Framework-maintenance release — no server behavior or tool-surface changes.

## Added

- **`scripts/check-dependency-specifiers.ts` + `Dependency Specifiers` devcheck step** — rejects floating specifiers (`latest`, `*`, pre-release dist-tags) in `package.json`'s dependency sections and `bun.lock`'s `workspaces` specifier maps. `bun update --latest` writes the literal `latest` dist-tag into the lock's workspace map even when `package.json` gets a caret range, which clears every other gate and lets the next `bun install` re-resolve past the range; this check catches that ([cyanheads/mcp-ts-core#246](https://github.com/cyanheads/mcp-ts-core/issues/246)). `peer`/`optional` sections may float to `*`/dist-tags; `latest` never passes anywhere. Shipped to consumers via `package.json` `files:` because `devcheck` invokes it.

## Changed

- **`@cyanheads/mcp-ts-core`** ^0.10.6 → ^0.10.9.
- **Packaging devcheck check** — now runs when `manifest.json` OR a plugin marketplace manifest (`.claude-plugin/plugin.json`, `.codex-plugin/plugin.json`, `.codex-plugin/mcp.json`) is present, validating the new manifests alongside `manifest.json`/`server.json` ([cyanheads/mcp-ts-core#240](https://github.com/cyanheads/mcp-ts-core/issues/240)); gated by `packaging.pluginManifests` in `devcheck.config.json`.
- **Git-dependent devcheck checks** — `TODOs/FIXMEs`, `Tracked Secrets`, and `Framework Antipatterns` skip cleanly when the project root has no `.git` (a fresh `init` scaffold before `git init`) instead of failing on `git grep`/`git ls-files` exit 128.
- **Skills synced from framework** — `add-tool`, `api-auth`, `api-canvas`, `api-config`, `api-context`, `api-errors`, `api-services`, `api-telemetry`, `field-test`, `git-wrapup`, `orchestrations`, `polish-docs-meta`, `report-issue-local`, and `tool-defs-analysis` refreshed; `api-context` documents `ctx.content`.
