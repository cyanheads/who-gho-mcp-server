# 0.1.12 — 2026-06-04

## Fixed

- **`normalizeRow`** in `gho-service.ts` no longer substitutes `year: 0` when `TimeDim` is absent; uses conditional spread so rows with a null `TimeDim` omit the field entirely ([#5](https://github.com/cyanheads/who-gho-mcp-server/issues/5)).
- **`DataRow.year`** in `types.ts` changed from `number` (required) to `number | undefined` (`year?: number`), matching the nullable upstream field.
- **`who_query_indicator_data` output schema** — `year` changed to `z.number().optional()` with an updated `.describe()` noting absence for time-independent rows; `format()` renders `**—**` when `year` is absent.
- **Tests** — added coverage for absent-year rows: handler passes without error, `format()` renders `**—**` placeholder, `undefined` does not appear in rendered output.
