# 0.1.13 — 2026-06-12

## Added

- **`createApp({ name, title })`** in `index.ts` — explicit `who-gho-mcp-server` identity on both fields, forwarded to the SDK `initialize` response and the server manifest (satisfies the framework's devcheck identity check).
- **`who_query_indicator_data` output** — `totalCount` (alias of `totalRows` for cross-tool consistency) and optional `truncated` boolean; the handler emits `ctx.enrich.truncated({ shown, cap, ceiling: 1000 })` when results are capped at the requested limit.
- **`scripts/clean-mcpb.ts`** — post-pack bundle cleaner wired into the `bundle` script; runs `mcpb clean` then strips dependency-shipped agent-doc entries (`node_modules/**/skills/`, `/.claude/`, `/.agents/`, stray `SKILL.md`) that root-anchored `.mcpbignore` patterns cannot reach, and asserts zero remain.
- **Dockerfile `HEALTHCHECK`** — polls `/healthz` every 30s via `bun -e fetch(...)`, no curl/wget dependency; plus `ARG APP_VERSION` and `org.opencontainers.image.version` OCI label.
- **`lint-packaging.ts` bundle-content checks** — new rules guard that `.mcpbignore` excludes root dev dirs (`KNOWN_DEV_DIRS`), uses `/`-anchored patterns, never strips `CRITICAL_RUNTIME_PATHS`, and that a built `.mcpb` contains zero `node_modules/**` agent-doc entries (`AGENT_DOC_ENTRY`, kept in sync with `clean-mcpb.ts`).
- **`check-framework-antipatterns.ts` rule 4** — flags `z.coerce.boolean()` on env flags across `src/` (`Boolean("false")` is `true`, so the flag can't be disabled); comment lines are skipped as mentions.

## Changed

- **`@cyanheads/mcp-ts-core`** ^0.9.21 → ^0.10.6.
- **`.mcpbignore`** — dev-dir patterns anchored to bundle root (`.claude/` → `/.claude/`, `.agents/` → `/.agents/`, `skills/` → `/skills/`) so they no longer also match nested `node_modules` paths.
- **Skills synced from framework** — `add-prompt`, `add-resource`, `add-test`, `add-tool`, `api-canvas`, `api-config`, `api-context`, `api-linter`, `api-mirror`, `api-testing`, `api-workers`, `report-issue-local`, `security-pass`, and others refreshed; new `techniques` skill (outline-on-overflow reference).
- **`CLAUDE.md`** — server identity section documents the `name`/`title` fields and the `z.stringbool()` env-boolean rule; dropped `ctx.sample` references (not part of the current context surface).
